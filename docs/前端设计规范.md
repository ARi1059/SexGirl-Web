## 设计决策摘要

| 维度 | 决定 |
|---|---|
| 视觉风格 | **杂志编辑风（Editorial）**：强排版、超大衬线标题、黑白高对比、锐利直角、错落网格 |
| 主题 | **双主题可切换**（亮 / 暗），首次访问跟随系统，手动切换后持久化 |
| 强调色 | **玫瑰粉 / 裸色**（唯一签名色，克制使用） |
| 关键字 | 精致 · 有态度 · 留白克制 · 图片为主角 · 单色点缀 |

## 1. 设计理念与原则

编辑风的本质是**用排版和图片说话，而不是用装饰**。落到本项目：

1. **图片是主角**：版面为商品图让路，大图、全出血（full-bleed）、克制的 UI 包裹。任何装饰都不能与图片抢视线。
2. **排版即设计**：层级靠字号/字重/字距的强对比建立，而非靠色块和阴影。超大衬线标题 + 干净无衬线正文。
3. **锐利、克制**：直角（圆角≈0）、发丝线（1px hairline）分隔、近乎无阴影。高级感来自留白与对齐，不是渐变和投影。
4. **单色点缀**：玫瑰粉/裸色是**唯一**彩色，只用于强调（CTA、眉标、状态点、hover）。其余皆黑白灰（暖调中性）。
5. **错落而有序**：画廊用编辑式错落网格（大小图交替、偶尔整幅 feature），但严格对齐基线栅格，乱中有序。
6. **双主题等价**：亮/暗两套都是一等公民，令牌成对定义，组件只引用语义变量，不写死颜色。

**Do / Don't**

| ✅ Do | ❌ Don't |
|---|---|
| 直角、发丝线、大留白 | 大圆角、卡片重投影、玻璃拟态 |
| 玫瑰粉只点缀 | 大面积粉色铺底、多种彩色并用 |
| 超大标题制造层级 | 靠加粗+变色堆叠层级 |
| 图片全出血、统一比例 | 图片加粗边框、随意比例 |
| uppercase 眉标 + 字距 | 正文也 uppercase、字距过宽影响中文 |

## 2. 设计令牌（Design Tokens）

所有令牌以 **CSS 变量**为源（`globals.css`），Tailwind 通过 `theme.extend` 引用（见 [§10](#10-落地映射)）。组件**只用语义变量**（如 `--bg` / `--accent`），不直接用 `--rose-500`。

### 2.1 颜色（亮 / 暗双主题）

暖调中性（不是纯灰，带一点暖/裸调）+ 玫瑰粉强调。

```css
/* globals.css —— 亮色（默认） */
:root {
  /* 中性 · 纸与墨 */
  --paper:        #FFFFFF;   /* 页面底 */
  --surface:      #FAF7F5;   /* 卡片/区块面（暖白） */
  --panel:        #F3ECE7;   /* 下沉面板/裸色块 */
  --ink:          #15110F;   /* 主文字（暖近黑） */
  --ink-muted:    #5C534D;   /* 次文字 */
  --ink-subtle:   #8A807A;   /* 三级/占位（仅大字或非关键） */
  --line:         #E8E1DC;   /* 发丝线/描边 */
  --line-strong:  #15110F;   /* 编辑式重规则线（极少用） */

  /* 强调 · 玫瑰粉 / 裸色 */
  --accent:        #C96A72;  /* 签名玫瑰粉：眉标/状态点/边框/大字 */
  --accent-strong: #B14E57;  /* 实心按钮(配白字)/hover/press —— 过 AA */
  --accent-soft:   #F8E8E9;  /* 浅粉底（标签/提示背景） */
  --on-accent:     #FFFFFF;  /* 实心强调上的文字 */
  --nude:          #E7D5C9;  /* 裸色：分隔/装饰块 */

  --focus:         var(--accent);
  --scrim:         linear-gradient(to top, rgba(0,0,0,.55), rgba(0,0,0,0) 60%);
  --shadow-overlay: 0 24px 60px -24px rgba(20,16,15,.35);
}

/* globals.css —— 暗色 */
.dark {
  --paper:        #131013;   /* 暖近黑（微微带紫） */
  --surface:      #1C181B;
  --panel:        #262126;
  --ink:          #F4F0EE;
  --ink-muted:    #B3A9A3;
  --ink-subtle:   #7E746F;
  --line:         #2C2730;
  --line-strong:  #F4F0EE;

  --accent:        #E6A0A7;  /* 暗底上提亮的玫瑰粉，对比更高 */
  --accent-strong: #EFB9BD;  /* hover 更亮 */
  --accent-soft:   #2A2024;  /* 暗粉底 */
  --on-accent:     #1A1416;  /* 实心粉按钮上用近黑字（对比≈8:1） */
  --nude:          #C9A99B;

  --focus:         var(--accent);
  --scrim:         linear-gradient(to top, rgba(0,0,0,.7), rgba(0,0,0,0) 60%);
  --shadow-overlay: 0 24px 60px -20px rgba(0,0,0,.6);
}
```

> **强调色用色铁律**：`--accent`（#C96A72）在白底对比约 **3.6:1**——只可用于**大字、图标、边框、状态点**，**不可作小号正文**。需要「粉底白字」的实心按钮一律用 `--accent-strong`（≈5.2:1，过 AA）。暗色主题上 `--accent` 已足够亮，实心按钮配近黑字 `--on-accent`。详见 [§8 对比度表](#8-可访问性accessibility)。

### 2.2 字体与排版

编辑风 = 高对比衬线大标题 + 干净无衬线正文；中文配思源宋/黑，等宽用于微信号/QQ号（便于辨识复制）。

```css
:root {
  --font-display: "Fraunces", "Noto Serif SC", Georgia, "Songti SC", serif;
  --font-sans:    "Inter", "Noto Sans SC", -apple-system, "PingFang SC", "Microsoft YaHei", sans-serif;
  --font-mono:    "Geist Mono", "SFMono-Regular", "JetBrains Mono", monospace;
}
```

**字号刻度（fluid，用 `clamp` 适配双端）**

| Token | 字号 (移动 → PC) | 行高 | 字重 | 字距 | 字体 | 用途 |
|---|---|---|---|---|---|---|
| `display-xl` | `clamp(2.75rem, 6vw, 5rem)` (44→80) | 1.02 | 600 | -0.02em | display | 首屏主标题 |
| `display-l` | `clamp(2rem, 4vw, 3.25rem)` (32→52) | 1.05 | 600 | -0.02em | display | 区块大标题 |
| `h1` | `clamp(1.75rem, 3vw, 2.5rem)` (28→40) | 1.1 | 600 | -0.01em | display | 详情页商品名 |
| `h2` | `1.5rem` (24) | 1.2 | 600 | -0.01em | display/sans | 次级标题 |
| `h3` | `1.25rem` (20) | 1.3 | 600 | 0 | sans | 小标题 |
| `body-l` | `1.125rem` (18) | 1.6 | 400 | 0 | sans | 引导正文 |
| `body` | `1rem` (16) | 1.6 | 400 | 0 | sans | 正文 |
| `small` | `0.875rem` (14) | 1.5 | 400 | 0 | sans | 辅助说明 |
| `overline` | `0.75rem` (12) | 1.4 | 600 | **0.16em** | sans | 眉标（uppercase / 中文加字距） |
| `mono` | `1rem` (16) | 1.4 | 500 | 0.02em | mono | 微信号 / QQ号展示 |

排版规则：标题用 display 衬线 + 负字距制造紧凑高级感；正文行高 1.6；**眉标（overline）**是编辑风灵魂——uppercase、强字距、常配玫瑰粉，用于「分类/状态/小节标识」。中文标题用思源宋（Noto Serif SC）。

### 2.3 间距与栅格节奏

4px 基准间距刻度：

| Token | px | Token | px |
|---|---|---|---|
| `space-1` | 4 | `space-6` | 32 |
| `space-2` | 8 | `space-7` | 48 |
| `space-3` | 12 | `space-8` | 64 |
| `space-4` | 16 | `space-9` | 96 |
| `space-5` | 24 | `space-10` | 128 |

**编辑式纵向节奏**：区块上下留白用 `clamp(64px, 10vw, 160px)`——编辑风偏好大开大合的呼吸感。卡片内边距 `space-5`（24）。

### 2.4 圆角・描边・阴影・层级

```css
:root {
  --r-0: 0px;      /* 卡片、图片、按钮 —— 默认锐利直角 */
  --r-1: 2px;      /* 弹层/输入框，极轻收边 */
  --border: 1px;   /* 发丝线 */
  --border-strong: 1.5px; /* 编辑式重规则线 */
}
```

- **圆角**：默认 `--r-0`（直角）。仅弹层/输入框允许 `--r-1`(2px)。**禁止**大圆角。
- **描边**：1px hairline（`--line`）是主要分隔手段，替代投影。
- **阴影**：UI **基本无阴影**（扁平）。仅 Modal/Drawer 用 `--shadow-overlay`。
- **层级 z-index**：`base 0` / `sticky header 100` / `dropdown 200` / `overlay/drawer 300` / `toast 400`。

### 2.5 动效令牌

```css
:root {
  --ease-out:   cubic-bezier(0.22, 1, 0.36, 1);   /* 入场/位移 */
  --ease-inout: cubic-bezier(0.65, 0, 0.35, 1);   /* 过渡 */
  --dur-fast:  120ms;   /* 微交互（hover、点击反馈） */
  --dur:       240ms;   /* 一般过渡 */
  --dur-slow:  480ms;   /* 列表↔详情、弹层 */
  --dur-image: 640ms;   /* 图片缩放/揭示 */
}
```

参数详见 [§7 动效细则](#7-动效细则)。

## 3. 双主题实现

技术：Tailwind `darkMode: 'class'`，在 `<html>` 上切换 `.dark` 类。

1. **首次访问**：跟随系统 `prefers-color-scheme`。
2. **手动切换**：写入 `localStorage('theme' = 'light'|'dark')`，覆盖系统偏好。
3. **防闪烁（FOUC）**：在 `<head>` 最前插入阻塞脚本，首帧前定好主题。

```tsx
// app/layout.tsx 的 <head> 内，置于最前（beforeInteractive）
<script
  dangerouslySetInnerHTML={{ __html: `
    (function(){try{
      var t=localStorage.getItem('theme');
      var d=window.matchMedia('(prefers-color-scheme: dark)').matches;
      if(t==='dark'||(!t&&d))document.documentElement.classList.add('dark');
    }catch(e){}})();
  `}}
/>
```

```ts
// 切换逻辑（主题切换按钮）
function toggleTheme() {
  const el = document.documentElement;
  const next = el.classList.toggle('dark') ? 'dark' : 'light';
  localStorage.setItem('theme', next);
}
```

- 组件**只引用语义变量**（`var(--bg)` 等），切换 `.dark` 即整站换肤，无需改组件。
- 切换按钮放页眉右侧（线性 太阳/月亮 图标），见 [§6.7](#67-页眉-masthead--页脚--主题切换)。
- 切换时给 `<html>` 加 `transition: background-color 240ms, color 240ms`（但 `prefers-reduced-motion` 下禁用）。

## 4. 栅格与布局

**断点**（Tailwind 默认）：

| 名 | 宽 | 画廊列数 | 说明 |
|---|---|---|---|
| 移动 | `< 768` | 1（偶尔全出血 feature） | 单列流，大图 |
| 平板 `md` | `768–1023` | 2 | 等宽双列 |
| 桌面 `lg` | `1024–1279` | 编辑式 2–3 列 + feature | 错落网格 |
| 大屏 `xl+` | `≥ 1280` | 同上，更宽边距 | 容器留白更大 |

**容器**：`max-width: 1440px`；左右边距 `clamp(20px, 5vw, 96px)`（编辑风宽边距）。

**画廊编辑式错落网格**（核心）：12 列栅格，普通卡跨 4（桌面）/6（平板），每第 5 项设为 feature 跨 8 或整幅 12；交替列做轻微纵向位移制造错落。

```css
.gallery {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  gap: var(--space-6);            /* 32 */
}
.gallery > .card        { grid-column: span 4; }            /* 桌面默认 */
.gallery > .card--feature { grid-column: span 8; }          /* 特写大图 */
@media (max-width:1023px){ .gallery>.card{grid-column:span 6;} .gallery>.card--feature{grid-column:span 12;} }
@media (max-width:767px){ .gallery>.card,.gallery>.card--feature{grid-column:span 12;} }
/* 错落：偶数列下移半步（仅桌面，reduced-motion 不影响布局） */
@media (min-width:1024px){ .gallery > .card:nth-child(odd){ transform: translateY(var(--space-7)); } }
```

> feature 卡可由后台 `sortOrder` 或一个「精选」开关驱动（如需，回到开发文档给 `products` 加 `featured` 字段）。

## 5. 图像规范

图片是主角，必须**比例统一、处理一致**。

| 用途 | 比例 | next/image 取尺寸 | 备注 |
|---|---|---|---|
| 列表卡封面 | **3:4 竖构图** | `card` 800w | 统一竖图最显编辑感 |
| feature 特写 | **4:5 或 16:9** | `full` 1600w | 整幅/大跨 |
| 详情大图 | 原图比例 | `full` 1600w | 轮播 |
| 二维码 | **1:1** | `thumbnail` 400w | 发丝线方框包裹 |

- **Payload `imageSizes`**：`thumbnail 400 / card 800 / full 1600`（对应开发文档 [§9.1](开发文档.md#L350)）。
- **格式**：`next/image` 自动 AVIF/WebP；懒加载；`placeholder="blur"` 模糊占位防 CLS。
- **叠字图（overlay）**：图上有文字时，底部加 `--scrim` 渐变蒙版保证可读，文字左下对齐 + 眉标。
- **hover 处理**：图片 `scale(1.04)` @ `--dur-image` `--ease-out`，容器 `overflow:hidden`。可选「灰度→彩色」揭示（编辑风常用），`prefers-reduced-motion` 下禁用。
- **alt**：必填（Payload `media.alt`），无障碍与 SEO。

## 6. 组件规格

组件落在 `src/components/`（`ui/` 基础件、`gallery/` 画廊、`renderers/` 分发件、`category/` 分类导航、`favorite/` 收藏、`account/` 账号，见开发文档 [§6](开发文档.md#L248)）。基础件基于 shadcn/ui，按本令牌覆写主题。

### 6.1 按钮

| 变体 | 视觉 | 用途 |
|---|---|---|
| **Primary** | 实心 `--accent-strong` 底 + `--on-accent` 字，直角，label uppercase + 字距 0.08em | 主 CTA「联系定制」 |
| **Secondary** | 透明底 + 1px `--ink` 边 + `--ink` 字 | 次操作 |
| **Ghost** | 无边无底，hover 显玫瑰粉下划线（draw 动效） | 文本链接 |

尺寸：`lg` 高 48 / `md` 高 40 / `sm` 高 32；内边距横向 `space-5`。圆角 `--r-0`。focus 见 §8。

### 6.2 眉标（Overline）与状态徽标

- **眉标 Overline**：`overline` 字号，uppercase（中文加 0.16em 字距），色 `--accent` 或 `--ink-muted`。用于小节标识、分类。
- **状态徽标**（不用色块，用「点 + 字」，避免只靠颜色传意）：
  - 「今日可接单」= `--accent` 实心圆点(6px) + 同色 uppercase 标签（文案取 `availableTodayText`，可自定义）。
  - 「已约满」= `--ink-subtle` 点 + 同色标签。
  - `statusText` 自定义文字 = 眉标样式渲染。
  - 「已下架」前台不展示（`published=false` 已过滤）。

### 6.3 标签 Tag（对应 4 类 Block）

呼应开发文档 [§5.2 TagRenderer](开发文档.md#L207)，按 `blockType` 给编辑式视觉：

| type | 视觉 |
|---|---|
| `text` | uppercase 小标签 + 1px 玫瑰粉下划线；`style(json)` 可覆写 color/bg |
| `icon` | 1.5px 线性图标（Lucide）+ 行内 label |
| `colorBlock` | **方形**色块(12px，取 `style` 色) + label（编辑风用方不用圆） |
| `link` | label + `↗`，hover 下划线，新标签打开 |

标签组横向排列、`space-2` 间隔，超出换行。

### 6.4 商品卡片（画廊）

```
┌───────────────────┐
│                   │   ← 封面 3:4 竖图，全出血，hover scale 1.04
│      [ 图片 ]      │
│                   │
├───────────────────┤   ← 1px 发丝线
│ ● 今日可接单        │   ← 眉标/状态（玫瑰粉点+字）
│ 商品标题            │   ← display 衬线，h3/h2
│ 标签 标签 标签       │   ← TagRenderer
└───────────────────┘
```

- 卡片无边框无投影，靠图片与发丝线分隔。
- hover：图缩放 + 标题显玫瑰粉下划线（draw）。
- 整卡可点 → `/p/[id]`，封面带 `view-transition-name`（见 §7）。

### 6.5 区块标题（编辑式 top-rule）

小节顶部一条 `--border-strong` 横规则线，线下并排：左 `display-l` 大标题，右 眉标/计数。营造杂志栏目感。

### 6.6 联系方式弹层（引流核心）

承接开发文档 [§5.3 交互表](开发文档.md#L233) 的视觉化。这是转化关键，要精致、好操作。

- **触发**：详情页 Primary 按钮「联系定制」/「立即咨询」。
- **形态**：桌面**居中 Modal**（max-w 420，直角，1px 边，`--surface` 底，`--shadow-overlay`）；移动**底部 Drawer** 上滑（全宽，顶部 `--r-1`）。
- **结构**：
  ```
  联系客服                      ← 眉标 overline
  小美 · 定制专属               ← display 标题(contact.label)
  ───────────────────────────  ← 发丝线
  [微信]  wx_abc123      [复制]  ← value 用 mono 字体；复制=Secondary 按钮
  [QQ]    123456    [复制][打开QQ] ← QQ 多一个 mqqwpa:// 深链按钮
  ┌─────┐
  │ QR  │  长按保存 / 扫码添加     ← 二维码 1:1 + 发丝线方框 + 说明
  └─────┘
  ```
- **多客服**：纵向列表，发丝线分隔；或「微信/QQ」分段切换。
- **复制反馈**：点复制 → `navigator.clipboard` → Toast「已复制，去微信添加」(玫瑰粉短提示，2s)。
- **动效**：Modal fade+scale(0.98→1) @ `--dur-slow`；Drawer slide-up @ `--dur-slow` `--ease-out`；背景蒙版 fade。

### 6.7 页眉 Masthead / 页脚 / 主题切换

- **页眉**：sticky，`--paper` 底 + 底部发丝线。左 = 站名 wordmark（display 衬线）；右 = 主题切换（线性 太阳/月亮）。编辑风刊头极简、靠字形。
- **主题切换**：图标按钮 40×40，点击切 `.dark` 并持久化（§3）；图标随主题切换淡入淡出。
- **页脚**：顶部发丝线，small 字号，版权 + 极简眉标链接。
- **账号入口**（M7）：刊头右侧、主题切换之左，放账号眉标链接（详见 §6.11）。登录态**仅在客户端读取**并据此切换：未登录显示「登录 / 注册」，已登录显示「个人中心 / 退出」。刊头本身保持 server 静态，登录态客户端水合，不破坏 ISR（开发文档 §7.4）。

### 6.8 分类导航条（Category Nav）

- **位置**：列表页 `/` 与各分类页 `/c/[slug]` 顶部、区块标题（§6.5）之上。
- **形态**：横向眉标条，上下各一道发丝线（`border-y` `--line`）；项用 `overline`（uppercase + 0.16em 字距），项间横向 `space-6`，移动端自动换行（不做横向滚动）。
- **构成**：`全部`（→ `/`）、`今日可接单`（恒为第二项 → 虚拟页 `/c/today`）、各商品类型（来自 `categories`，按 `sortOrder`）。
- **态**：当前项 `--accent` + draw 下划线（留存）；其余 `--ink-muted`，hover `--ink` + draw 下划线。当前项标 `aria-current="page"`。
- **数据**：分类由 server 页面读取后以 props 传入；仅「当前项高亮」需客户端 `usePathname`。

### 6.9 分类页（`/c/[slug]` 与今日可接单 `/c/today`）

- **组合**：顶部 §6.8 分类导航 + §6.5 区块标题（top-rule）+ §6.4 画廊网格，与列表页同构。
- **标题区**：左 `display-l` 大标题 = 分类名；**今日可接单页**标题取商品的 `availableTodayText`（缺省「今日可接单」）。右 `overline` 计数 = `{N} 款 · {分类名 / 今日可接单}`。
- **空状态**：克制一句（§9），不堆砌。
- **缓存**：与列表/详情同走 ISR（不影响视觉）；分类增改删与商品归类/可接单变动即时 revalidate（开发文档 §8.5）。

### 6.10 收藏按钮（Favorite Button）

- **形态**：心形图标按钮（Lucide `Heart`，1.5px 线性描边）。**遵守直角铁律**：不用圆形，用 `--r-1`(2px) 极轻收边（浮层图标按钮，归 §2.4 允许轻收边之列）。
- **态**：未收藏 = 描边心形 `--ink`，hover `--accent`；已收藏 = `--accent` 实心填充（fill + stroke 同色）。
- **位置**：
  - **画廊卡**：绝对定位封面右上（`space-3` 内缩，36×36），半透明 `--paper`/70 + backdrop-blur + 1px 发丝线，保证任意封面上可读；作为整卡 `<Link>` 的**同级**覆盖（z-10），点击不触发跳转（杜绝 `<button>` 嵌 `<a>`）。
  - **详情页**：商品名右侧，44×44，`--paper` 实底 + 发丝线。
- **反馈**：点按 `scale(0.9)` @ `--dur-fast`（`prefers-reduced-motion` 下关闭）；乐观更新即时翻转填充，失败回滚。
- **未登录**：点击跳 `/login?redirect=<当前页>`，登录后回到原页。
- **无障碍**：`aria-pressed` 反映收藏态；`aria-label`「收藏」/「取消收藏」；hit area ≥ 36（卡）/ 44（详情）。

### 6.11 账号表单（登录 / 注册）

- **页面**：`/login`、`/register`，编辑风 header（`overline`「会员」+ `display-l`「登录」/「注册」；注册页附一句 `body` 引导）。
- **表单**：单列，`max-width 420`。字段 label 用 `overline`；输入框 1px 发丝线 + `--paper` 底 + `--r-1` 轻收边 + 内距（横 `space-4` / 纵 `space-3`），`focus` 时 `--accent` 边 + focus 环（§8）。
- **提交**：整宽 **Primary** 按钮（§6.1，实心 `--accent-strong` + `--on-accent`）；进行中文案「处理中…」。
- **错误**：表单内一行 `--accent-strong` `small` 文字、`role="alert"`（不弹 toast、不只靠颜色）。
- **切换**：底部 **Ghost** 文本链接「去登录 / 去注册」。
- **字段**：用户名（必填）、密码（注册 ≥ 6 位）、昵称（注册可选）。最小可用，无邮箱 / 找回密码（开发文档 §7.5 / 决策 #8）。
- **行为**：注册成功即自动登录并整页跳转 `redirect`（默认 `/me`），让登录态重新水合。

### 6.12 个人中心（`/me`）

```
个人中心                          ← 眉标 overline           [退出登录] ← Ghost
{昵称} 的收藏                     ← display-l 大标题
N 款已收藏                        ← 眉标计数
─────────────────────────────────  （区块标题 top-rule §6.5）
[  收藏商品网格 · 复用画廊卡片  ]   ← §6.4，可在卡上直接取消收藏
```

- **可见性**：仅登录可见，未登录跳 `/login`；每用户**动态**渲染（`force-dynamic`），不进 CDN / ISR（开发文档 §7.6）。
- **主体**：复用 §6.4 画廊卡片网格展示收藏商品（卡上收藏按钮可直接取消）。
- **空状态**：「还没有收藏。回画廊点心形即可收藏心仪款式。」（§9）。

## 7. 动效细则

统一用开发文档选定的 **Motion (Framer Motion) + View Transitions**，参数取 §2.5 令牌。

- **入场（画廊卡）**：`opacity 0→1` + `translateY(20px→0)`，`--dur-slow` `--ease-out`，**stagger 70ms** 逐个。进入视口触发（once）。
- **列表 ↔ 详情**：**View Transitions** 共享元素——封面图设 `view-transition-name: product-{id}`，根过渡 `--dur-slow` `--ease-out`，实现图片「飞入」详情。
- **hover**：图片 `scale(1.04)` @ `--dur-image`；链接/标题 下划线 draw（width 0→100%）@ `--dur`；按钮 底色 `--dur-fast`。
- **弹层**：见 §6.6。
- **主题切换**：背景/文字色 240ms 过渡。
- **`prefers-reduced-motion: reduce`**：**全部位移/缩放禁用**，仅保留 `opacity`；过渡时长降到 1ms；错落布局位移保留（布局非动画）。

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after { animation-duration:.001ms!important; transition-duration:.001ms!important; }
  /* transform 类动效在组件层用 useReducedMotion() 关掉 */
}
```

## 8. 可访问性（Accessibility）

**目标 WCAG 2.1 AA。**

**对比度（关键配对，约值）**

| 前景 / 背景 | 亮色 | 暗色 | 结论 |
|---|---|---|---|
| `--ink` / `--paper` | ~19:1 | ~16:1 | ✅ 正文 |
| `--ink-muted` / `--paper` | ~7.5:1 | ~7:1 | ✅ 次文字 |
| `--ink-subtle` / `--paper` | ~3.9:1 | ~4:1 | ⚠️ 仅大字/非关键 |
| `--accent` / `--paper` | ~3.6:1 | ~8.7:1 | 亮色仅大字/图标/边框；暗色可文字 |
| `--on-accent` / `--accent-strong`（按钮） | ~5.2:1 | ~8:1 | ✅ 实心按钮文字 |

- **焦点可见**：所有可交互元素 `:focus-visible` → 2px `--focus` 实线 + 2px 偏移；绝不 `outline:none` 无替代。
- **不只靠颜色**：状态用「点 + 文字」（§6.2），链接 hover 有下划线。
- **触达尺寸**：可点目标 ≥ 44×44px（移动）。
- **语义**：标题层级有序（一个 h1/页）；图片必有 `alt`；弹层焦点陷阱 + Esc 关闭 + 返回触发元素。
- **表单**（账号）：label 关联 input（`for`/`id`）；错误用行内 `role="alert"` 文本（不只靠颜色）。收藏按钮用 `aria-pressed` 反映状态。
- **动效**：尊重 `prefers-reduced-motion`（§7）。

## 9. 文案与语气

- **调性**：自信、简练、编辑感。中文为主（决策 #3）。
- **眉标**：可用 uppercase 拼音/英文或加字距中文（如 `今 日 上 新`）。
- **CTA**：动词开头、短——「联系定制」「立即咨询」「复制微信号」「扫码添加」。
- **状态**：「今日可接单」「已约满」「定制中」——简短、与 `availableTodayText` / `statusText` 一致。
- **账号 / 收藏**（M7）：「登录」「注册」「注册并登录」「个人中心」「退出」；收藏按钮 `aria-label` 用「收藏 / 取消收藏」；注册引导一句「自设用户名与密码即可收藏心仪商品」。
- **空状态**：克制一句 + 一个 CTA，不堆砌（如个人中心无收藏：「还没有收藏。回画廊点心形即可收藏心仪款式。」）。
